# Test Plan - COBOL Student Account Application

This test plan documents the current business logic and implementation behavior of the COBOL account management application. It is intended for validation with business stakeholders and as a foundation for future unit and integration tests in a Node.js rewrite.

Observe the response | The system displays an invalid choice message and returns to the menu loop | TBD | Not Run | Covers menu validation in MainProgram |

## Notes

- The current COBOL implementation uses a single runtime balance, not multiple student records.
- Input validation is limited; the application does not currently enforce richer business rules such as minimum deposit amounts or non-negative checks beyond the debit sufficiency rule.
- This table can be reused later to derive Node.js unit tests, integration tests, and stakeholder acceptance scenarios.
